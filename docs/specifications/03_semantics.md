# 03. Семантика языка

## 1. Типы данных

### 1.1. Встроенные типы

Язык поддерживает пять встроенных типов данных:

| Тип | Описание | Размер | Диапазон / Значения |
|-----|----------|--------|---------------------|
| `Int` | Целое число со знаком | 32 бита | −2³¹ … 2³¹−1 (−2147483648 … 2147483647) |
| `Float` | Число с плавающей точкой | 64 бита | IEEE 754 double precision |
| `Bool` | Логический тип | 8 бит | `true`, `false` |
| `String` | Неизменяемая последовательность символов | переменный | UTF-16 кодировка |
| `Void` | Отсутствие значения | — | Используется только как тип возврата функции |

### 1.2. Особенности типов

#### Int
- Арифметические операции: `+`, `-`, `*`, `/`, `%`
- Деление `/` между `Int` — целочисленное (дробная часть отбрасывается)
- Оператор `%` (остаток) доступен только для `Int`

#### Float
- Арифметические операции: `+`, `-`, `*`, `/`, `**`
- Оператор `%` недоступен для `Float`
- Сравнение `Float` на равенство (`==`, `!=`) допускается, но не рекомендуется из-за погрешностей

#### Bool
- Логические операции: `!`, `&&`, `||`
- Неявное преобразование между `Bool` и числовыми типами **запрещено**
- В условиях (`if`, `while`, `for`) требуется выражение типа `Bool`

#### String
- Неизменяемый тип: любая операция создаёт новую строку
- Операция `+` выполняет конкатенацию строк
- Сравнение строк (`==`, `!=`, `<`, `<=`, `>`, `>=`) выполняется лексикографически

#### Void
- Не может быть типом переменной или параметра
- Используется только как тип возврата функции, не возвращающей значение
- Функция типа `Void` может опускать `return` или использовать `return;` без выражения

### 1.3. Преобразования типов

#### Неявные преобразования
**Запрещены.** Все преобразования между типами должны выполняться явно.

## 2. Области видимости и время жизни

### 2.1. Лексическая область видимости

Используется статическая (лексическая) область видимости:
- Идентификатор разрешается в ближайшей охватывающей области, где он объявлен

### 2.2. Уровни областей видимости

| Уровень | Описание | Доступные сущности |
|---------|----------|-------------------|
| Глобальная | Весь файл программы | `const`, `func`, `var`, `let` на верхнем уровне |
| Функции | Тело функции | Параметры, локальные переменные |
| Блока | Внутри `{ }` | Переменные, объявленные в блоке |

### 2.3. Правила видимости

- **Уникальность имён**: в пределах одной области видимости имена должны быть уникальны
- **Запрет сокрытия (shadowing)**: нельзя объявить локальную переменную с именем, уже используемым во внешней области
- **Порядок объявления**: функции могут объявляться в любом порядке (взаимная рекурсия поддерживается); переменные должны быть объявлены до использования

### 2.4. Время жизни

| Сущность | Время создания | Время уничтожения |
|----------|---------------|-------------------|
| Глобальная переменная | До начала выполнения программы | После завершения программы |
| Локальная переменная | При входе в блок | При выходе из блока |
| Параметр функции | При вызове функции | При завершении функции |
| Константа | На этапе компиляции | Не применимо (подставляется в код) |

## 3. Семантика выражений

### 3.1. Порядок вычисления

- Аргументы функций вычисляются **слева направо**
- Операнды бинарных операторов вычисляются **слева направо**
- Исключение: операторы `&&`, `||`, `?:` используют ленивое вычисление

### 3.2. Ленивое вычисление (Short-circuit Evaluation)

| Оператор | Правило |
|----------|---------|
| `a && b` | Если `a == false`, `b` не вычисляется |
| `a \|\| b` | Если `a == true`, `b` не вычисляется |
| `a ? b : c` | Вычисляется только `b` или только `c` в зависимости от `a` |

### 3.3. Типизация операторов

#### Арифметические операторы

| Оператор | Операнды | Результат | Примечание |
|----------|---------|-----------|------------|
| `+`, `-`, `*` | `Int`, `Int` | `Int` | — |
| `+`, `-`, `*` | `Float`, `Float` | `Float` | — |
| `+`, `-`, `*` | `Int`, `Float` | `Float` | `Int` неявно преобразуется |
| `/` | `Int`, `Int` | `Int` | Целочисленное деление |
| `/` | `Float`, `Float` | `Float` | — |
| `%` | `Int`, `Int` | `Int` | Только для `Int` |
| `**` | `Float`, `Float` | `Float` | Возведение в степень |
| `+` | `String`, любой | `String` | Конкатенация |

#### Операторы сравнения

| Оператор | Операнды | Результат |
|----------|---------|-----------|
| `==`, `!=` | Любые совместимые типы | `Bool` |
| `<`, `<=`, `>`, `>=` | `Int`, `Float`, `String` | `Bool` |

> Сравнение `String` выполняется лексикографически по кодам Unicode.

#### Логические операторы

| Оператор | Операнд | Результат |
|----------|---------|-----------|
| `!` | `Bool` | `Bool` |
| `&&`, `\|\|` | `Bool`, `Bool` | `Bool` |

### 3.4. Присваивание

- Левая часть должна быть изменяемой переменной (`var`)
- Присваивание константе (`const`) или неизменяемой переменной (`let`) — ошибка компиляции
- Тип правой части должен быть совместим с типом левой части (с учётом неявных преобразований)

## 4. Семантика управляющих конструкций

### 4.1. Условный оператор `if`

```
if (condition) {
    // выполняется если condition == true
} else {
    // выполняется если condition == false
}
```

**Правила:**
- `condition` должно иметь тип `Bool`
- Использование числовых типов в условии запрещено (нет неявного преобразования 0 → false)
- Ветвь `else` опциональна

### 4.2. Тернарный оператор `?:`

```
result = condition ? trueExpr : falseExpr;
```

**Правила:**
- `condition` должно иметь тип `Bool`
- Вычисляется только одна из ветвей (`trueExpr` или `falseExpr`)
- Типы ветвей должны быть совместимы (результат — общий тип)
- Правоассоциативен: `a ? b : c ? d : e` ≡ `a ? b : (c ? d : e)`

### 4.3. Цикл `while`

```
while (condition) {
    // тело цикла
}
```

**Правила:**
- `condition` должно иметь тип `Bool`
- Проверка условия происходит **перед** каждой итерацией
- Если `condition == false` при входе, тело не выполняется ни разу

### 4.4. Цикл `for`

```
for (initialization; expression; expression) {
    // тело цикла
}
```

**Правила:**
- Все три части опциональны
- Пропуск `condition` эквивалентен `true` (бесконечный цикл)
- Переменная, объявленная в `initialization`, видна только внутри цикла

### 4.5. Функции

**Вызов функции:**
1. Вычисляются аргументы слева направо
2. Значения аргументов копируются в параметры (передача по значению)
3. Выполняется тело функции
4. При `return` управление возвращается вызывающему коду

**Правила:**
- Параметры передаются **по значению** (изменение параметра не влияет на аргумент)
- **Замыкания не поддерживаются**: функция не захватывает локальные переменные внешней области
- Рекурсия поддерживается
- Функция с типом возврата, отличным от `Void`, должна содержать путь выполнения с `return`

#### Встроенные функции

| Функция | Сигнатура | Описание | Ошибка выполнения |
|---------|-----------|----------|------------------|
| `abs` | `Float abs(Float x)` | Модуль числа | — |
| `min` | `Float min(Float a, Float b, ...)` | Минимум из аргументов | — |
| `max` | `Float max(Float a, Float b, ...)` | Максимум из аргументов | — |
| `toInt` | `Int toInt(String s)` | Преобразование строки в `Int` | Некорректный формат |
| `toFloat` | `Float toFloat(String s)` | Преобразование строки в `Float` | Некорректный формат |

> Встроенные функции `min`, `max` принимают два и более аргументов типа `Float`.

## 5. Константы и переменные

### 5.1. Константы (`const`)

| Характеристика | Описание |
|---------------|----------|
| Время вычисления | Компиляция |
| Инициализация | Обязательна при объявлении |
| Выражение | Только константное (литералы, другие `const`) |
| Память | Не выделяется (подставляется в код) |
| Изменяемость | Запрещена |

**Пример:**
```
const Pi: Float = 3.14159;
const MaxSize: Int = 100;
```

### 5.2. Неизменяемые переменные (`let`)

| Характеристика | Описание |
|---------------|----------|
| Время вычисления | Выполнение |
| Инициализация | Обязательна при объявлении |
| Выражение | Любое допустимое выражение |
| Память | Выделяется (имеет адрес) |
| Изменяемость | Запрещена после инициализации |

**Пример:**
```
let size: Int = GetValue();  // вызов функции допустим
let half: Float = Pi / 2;    // использование const допустимо
```

### 5.3. Изменяемые переменные (`var`)

| Характеристика | Описание |
|---------------|----------|
| Время вычисления | Выполнение |
| Инициализация | Опциональна (но использование до инициализации запрещено) |
| Выражение | Любое допустимое выражение |
| Память | Выделяется (имеет адрес) |
| Изменяемость | Разрешена |

**Пример:**
```
var counter: Int;      // объявление без инициализации
counter = 0;           // инициализация перед использованием
counter = counter + 1; // изменение значения
```

## 6. Ввод и вывод

### 6.1. Вывод (`print`)

```
print(expression);
```

**Правила:**
- Выражение вычисляется
- Результат преобразуется в строку
- Строка выводится в стандартный поток вывода
- Перевод строки не добавляется автоматически

### 6.2. Ввод (`input`)

```
input(variable);
```

**Правила:**
- Тип вводимого значения должен соответствовать типу переменной
- При некорректном вводе возникает ошибка выполнения
- Читается одно слово с клавиатуры (последовательность символов без пробелов, табуляций, перевода строки)
- Чтение прекращается при встрече первого пробельного символа
- Ведущие и завершающие пробелы игнорируются

## 7. Контекстно-зависимые правила

### 7.1. Объявление и использование

| Правило | Ошибка |
|---------|--------|
| Переменная должна быть объявлена до использования | «Необъявленный идентификатор» |
| Повторное объявление имени в одной области | «Повторное объявление» |
| Использование `const`/`let` в левой части присваивания | «Присваивание неизменяемой сущности» |

### 7.2. Типизация

| Правило | Ошибка |
|---------|--------|
| Условие `if`/`while`/`for` должно быть `Bool` | «Ожидается Bool» |
| Операнды `%` должны быть `Int` | «Оператор % требует Int» |
| Аргументы функции должны соответствовать параметрам | «Несоответствие типов аргументов» |
| `return` должен соответствовать типу функции | «Несоответствие типа возврата» |

### 7.3. Инициализация

| Правило | Ошибка |
|---------|--------|
| `const` без инициализации | «Константа требует инициализации» |
| `let` без инициализации | «let требует инициализации» |
| Использование `var` до инициализации | «Использование неинициализированной переменной» |

### 7.4. Функции

| Правило | Ошибка |
|---------|--------|
| Число аргументов ≠ числу параметров | «Несоответствие числа аргументов» |
| Функция без `return` (тип не `Void`) | «Отсутствует возврат значения» |
| Рекурсия без условия завершения | «Переполнение стека» (ошибка выполнения) |


## 8. Требования к семантически корректным программам

1. **Полнота объявлений**: все используемые идентификаторы должны быть объявлены
2. **Уникальность имён**: в каждой области видимости имена уникальны
3. **Совместимость типов**: все операции выполняются над совместимыми типами
4. **Инициализация**: все переменные инициализированы до использования
5. **Возврат значений**: функции с типом возврата имеют путь выполнения с `return`
6. **Корректность условий**: все условия имеют тип `Bool`
7. **Отсутствие сокрытия**: имена не скрывают имена из внешних областей
